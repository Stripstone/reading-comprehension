Actually worth flagging — there is no decay happening by default. The files are written with CacheControl: "public, max-age=31536000, immutable" which tells browsers and CDNs how long to cache the URL, but that doesn't affect S3 storage itself. S3 keeps objects forever until explicitly deleted. A page synthesized today will still be sitting in the bucket in 5 years if nothing removes it.

The cost is very low — at ~250KB per page and $0.023/GB/month, 10,000 unique pages costs about $0.06/month in storage. So it's not urgent, but if you want actual decay you'd set an S3 Lifecycle Policy.

The right policy for this use case would be something like: delete objects in the tts/ prefix that haven't been accessed in 90 days. This requires enabling S3 Last Accessed tracking (which is off by default) alongside the lifecycle rule.

A simpler alternative that doesn't require access tracking: just delete everything older than 1 year.
